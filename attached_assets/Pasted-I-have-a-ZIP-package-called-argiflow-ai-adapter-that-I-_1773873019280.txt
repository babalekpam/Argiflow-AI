I have a ZIP package called argiflow-ai-adapter that I need you to integrate into this project. Here is exactly what it contains and what to do with each file:

OVERVIEW
This is a universal AI provider adapter for ArgiFlow. It replaces all direct @anthropic-ai/sdk calls throughout the codebase with a single unified ai.chat() function that works with any AI provider. OpenAI (gpt-4o) is the default provider.

FILES AND INSTRUCTIONS
1. argiflow-ai-adapter/index.js
This is the core adapter. Place it at the root of the project as argiflow-ai-adapter/index.js. It exports:

ai.chat(input, options) — main function, accepts a string or message array
ai.chatWithSystem(system, message, options) — convenience wrapper with system prompt
ai.chatJSON(input, options) — same as chat but auto-parses JSON response
ai.listProviders() — returns all supported providers and which are configured
ai.getActiveProvider() / ai.getActiveModel() — returns current config

It reads these environment variables from Replit Secrets:

AI_PROVIDER — which provider to use (default: openai)
AI_MODEL — model override (default: gpt-4o for OpenAI)
OPENAI_API_KEY — required for OpenAI
ANTHROPIC_API_KEY — required for Anthropic
GEMINI_API_KEY, MISTRAL_API_KEY, GROQ_API_KEY, COHERE_API_KEY, TOGETHER_API_KEY, OPENROUTER_API_KEY — optional, for other providers
OLLAMA_BASE_URL — for local Ollama (default: http://localhost:11434)

No npm packages are required for this file — it uses Node.js built-in https/http modules only.

2. argiflow-ai-adapter/aiProviderRoutes.js
An Express router. Mount it in server.js like this:
jsconst aiRoutes = require('./argiflow-ai-adapter/aiProviderRoutes');
app.use('/api/ai', aiRoutes);
It adds these endpoints:

GET /api/ai/providers — lists all providers and whether they are configured
POST /api/ai/test — sends a test message to verify the active provider works
PATCH /api/ai/provider — switches the active provider at runtime (body: { provider, model })
